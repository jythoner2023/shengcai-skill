# Ultimate Template

This reference merges:

- `龙珠修行营`: value discovery and writer preparation
- `龙珠精华文章写作框架List`: the classic result-process-problem-solution skeleton
- the large-sample 生财精华 analysis: title formulas, opening rhythm, paragraph sequencing, and closing patterns

Use the sections below in order:

1. 选题
2. 结构
3. 标题
4. 开头
5. 中段
6. 金句
7. 结尾

## Rewrite Existing Markdown Draft

This skill should usually be applied to an existing Markdown draft, not a blank page.

Use this rewrite sequence:

1. Extract the non-negotiable facts from the draft.
2. Mark weak parts:
   - throat-clearing
   - repetition
   - vague claims
   - unsupported praise
   - unordered notes
3. Decide the primary article type.
4. Re-map the draft into the Shengcai structure.
5. Rewrite in place:
   - stronger title
   - stronger opening
   - tighter numbered middle
   - clearer closing
6. Save as a new Markdown file by default.
7. Keep the output in Markdown.

Hard constraints:

- Do not fabricate data, screenshots, customer quotes, or outcomes.
- Do not silently change core facts such as dates, prices, revenue, counts, names, or tools.
- If a claim has no support, weaken it or remove it.
- Prefer shortening and reorganizing before expanding.
- If the original has a good line, keep it and improve its placement.
- Preserve the original draft unless the user explicitly asks to overwrite it.
- Default output name: `<original-stem>.shengcai.md`.

Quick rewrite checklist:

- Can the first two paragraphs survive on their own?
- Does the middle move from problem to action to evidence to lesson?
- Is every big claim backed by something observable?
- Does the ending tell the reader what to do with the story?
- Are relative image paths and links still valid in the new sibling file?

## 1. 选题

### 1.1 先判断这篇文章值不值得写

Only write if the material has at least two of the following:

- a concrete result
- a concrete process
- a non-obvious lesson
- a visible constraint, mistake, or turning point
- a clear audience who can use it

If the material is only “我学到了很多”, it is not enough.

### 1.2 先把价值点显性化

Answer these in bullet form before drafting:

- 我做了什么事情？
- 我拿到了什么结果？
- 我为什么能做到？
- 我踩过什么坑？
- 哪些经验别人拿去就能用？
- 这篇文章最适合帮助谁？
- 如果只允许读者记住一句话，那句话是什么？

### 1.3 选题角度决策

Use one primary angle:

- `结果复盘型`
  - 适合：有收入、增长、项目跑通、客户成交、产品上线、数据提升
  - 承诺句：我做成了什么，以及我是怎么做成的
- `方法教程型`
  - 适合：有稳定方法、SOP、工作流、提示词、工具链
  - 承诺句：这件事怎么做，按什么顺序做
- `趋势判断型`
  - 适合：你看到了平台变化、供需变化、行业窗口
  - 承诺句：为什么现在值得做，普通人怎么切进去
- `认知升级型`
  - 适合：你最重要的价值是重定义问题
  - 承诺句：大家以为是X，其实关键是Y
- `记录成长型`
  - 适合：你从不会到会、从迷茫到跑通
  - 承诺句：我是怎么走过这段路的，哪些地方你可以少踩坑

### 1.4 生财读者过滤器

把选题改写成对生财读者有用的话：

- 不要写：`我最近做了一个项目`
- 要写：`普通人如何用这个项目拿到第一笔结果`

- 不要写：`我对AI有一些思考`
- 要写：`AI时代这类人真正该抓住什么机会`

- 不要写：`我有一些写作心得`
- 要写：`如何把经验写成能带来信任和机会的文章`

## 2. 结构

### 2.1 通用终极骨架

这是默认骨架，适用于绝大多数生财精华帖：

1. `标题`
2. `第一段抢注意力`
3. `第二段交代为什么值得看`
4. `背景 / 起点 / 问题`
5. `关键转折`
6. `关键动作 1`
7. `关键动作 2`
8. `踩坑 / 修正 / 失败经验`
9. `结果验证`
10. `抽象成方法`
11. `给读者的建议 / 下一步行动`

### 2.2 结果复盘型结构

```text
标题
开头先报结果
我是谁 + 为什么这件事值得说
当时遇到的问题
我是怎么开始的
做对了哪几件事
踩了哪些坑
最后结果如何
复盘：如果重来我会怎么做
对读者的建议
```

### 2.3 方法教程型结构

```text
标题
这篇文章适合谁 / 不适合谁
先说最终目标
为什么多数人会卡住
步骤 1
步骤 2
步骤 3
常见错误
低成本起步版本
总结：先做什么，再做什么
```

### 2.4 趋势判断型结构

```text
标题
先给判断
为什么很多人低估了它
平台 / 市场 / 供给 / 需求发生了什么变化
案例证明这个判断不是空话
现在下场的人有什么优势
普通人怎么切入
哪些人不适合
结论 + 行动建议
```

### 2.5 每一节的微结构

每个主体段最好遵循这个最小单元：

`判断句 -> 解释句 -> 例子/数据 -> 小结句`

例子：

- 判断句：真正让我跑通的，不是工具本身，而是先把需求掰开。
- 解释句：之前我总想一步做完整，结果每次都卡死在复杂度上。
- 例子/数据：改成先做最小版本后，两天内就拿到了第一个可用结果。
- 小结句：所以第一步不是堆功能，而是先跑通最小闭环。

## 3. 标题

### 3.1 标题的四个关键槽位

一个生财风格强标题，优先塞这四类信息：

- 结果
- 周期
- 对象或场景
- 方法或复盘属性

常见组合：

- `数字 + 方法`
- `数字 + 复盘`
- `数字 + 结果`
- `结果 + 对象 + 复盘`

### 3.2 直接套用的标题公式

#### 结果复盘型

- `X天 / X个月 + 从A到B：我如何做到Y`
- `不投流 / 不懂技术 / 0基础，如何做到Y`
- `从0到X，我做对了什么，踩过什么坑`
- `我把X做到了Y，完整复盘`

#### 方法教程型

- `如何用X做成Y`
- `普通人怎么从0开始做X`
- `X的完整SOP / 工作流 / 教程`
- `手把手拆解：如何做到Y`

#### 趋势判断型

- `为什么我现在看好X`
- `X是不是一个值得做的方向？`
- `我为什么判断X还有机会`
- `平台正在发生什么变化，普通人怎么接住`

#### 认知升级型

- `你以为X，实际上关键在Y`
- `真正决定X的，不是A，而是B`
- `很多人低估了X`
- `这件事表面看是X，本质上是Y`

### 3.3 标题约束

- 结果词尽量往前放。
- 数字只写真实数字，不要虚夸。
- 标题不是口号，必须让读者知道能得到什么。
- 允许长，但不能空。
- 如果标题很长，后半句必须补“为什么值得看”。

### 3.4 标题自检

检查这四个问题：

1. 看完标题，读者知道文章讲什么吗？
2. 看完标题，读者知道为什么值得点开吗？
3. 结果、对象、方法至少出现了两个吗？
4. 把“分享”“一些思考”“我的经历”删掉后，标题是不是更强？

## 4. 开头

### 4.1 开头的目标

前两段只做四件事：

- 抢注意力
- 建立可信度
- 告诉读者这篇对谁有用
- 把人送进正文

### 4.2 四种高频开头

#### 结果开头

适合：你有明确结果。

模板：

```text
先说结果：我在[周期]内做成了[结果]。
这篇文章，我会完整讲清楚我是怎么做到的，哪些动作最关键，哪些坑最容易踩。
```

#### 身份开头

适合：你的人设反差强，比如小白、边缘人、上班族、跨行者。

模板：

```text
大家好，我是[身份]。
如果你和我一样[处境 / 起点限制]，这篇文章可能会帮你少走很多弯路。
```

#### 问题开头

适合：读者痛点强。

模板：

```text
很多人做X的时候，卡住的并不是[表面问题]，而是[真正问题]。
我也是在反复踩坑之后，才意识到这件事真正该怎么做。
```

#### 判断开头

适合：趋势和认知类文章。

模板：

```text
我现在越来越确定，[判断句]。
很多人之所以没看到这个机会，不是因为它不存在，而是因为他们还在用旧视角看它。
```

### 4.3 开头禁忌

- 不要一上来讲大而空的背景。
- 不要先铺概念，后给结果。
- 不要自我介绍太长。
- 不要第一段就开始长篇教学。

## 5. 中段

### 5.1 中段的主要任务

中段负责三件事：

- 讲清你做了什么
- 讲清为什么有效
- 讲清别人能怎么用

### 5.2 中段最稳的推进顺序

推荐顺序：

1. 背景和起点
2. 当时卡住的问题
3. 关键转折
4. 做对的动作
5. 踩坑与修正
6. 数据和结果
7. 方法抽象

### 5.3 中段的强写法

- 用编号小节，降低阅读压力。
- 每节只讲一个核心点。
- 每两三节至少给一次证据：
  - 数据
  - 截图
  - 客户反馈
  - before/after 对比
  - 真实对话
- 重要判断最好落一句结论句。

### 5.4 中段的弱写法

- 只有经历，没有抽象。
- 只有观点，没有案例。
- 只有结果，没有过程。
- 只有过程，没有判断。
- 只有“我觉得”，没有数据和细节。

### 5.5 中段填空提纲

把下面问题按顺序写成小标题：

- 我原来处于什么状态？
- 我最开始遇到的核心问题是什么？
- 我为什么决定这样做？
- 我做的第一步是什么？
- 我做的第二步是什么？
- 哪个地方最容易失败？
- 我是怎么发现问题并修正的？
- 最后结果如何？
- 这件事背后的核心规律是什么？

## 6. 金句

### 6.1 金句的作用

生财风格里的金句，不是文艺句，而是“可复用的判断句”。
它的任务是把局部经验，压缩成读者能记住的规则。

### 6.2 高频句型

#### 不是X，而是Y

用途：重定义问题。

模板：

- 这件事真正难的，不是X，而是Y。
- 大多数人以为X，但真正决定结果的是Y。

#### 真正 / 核心 / 本质

用途：把经验升维成规律。

模板：

- 真正拉开差距的，是X。
- 核心不是怎么做得更复杂，而是怎么先跑起来。
- 这件事的本质，是Y。

#### 从X到Y

用途：写成长弧线和升级路径。

模板：

- 我是怎么从X走到Y的。
- 这件事最大的变化，是从X转向了Y。

#### 如果X，就Y

用途：给出条件判断。

模板：

- 如果你现在处于X阶段，先做Y。
- 如果你还没有结果，不要急着做Y，先补X。

#### 先X，再Y

用途：降低执行难度。

模板：

- 先跑通最小版本，再去优化。
- 先验证需求，再考虑放大。

### 6.3 金句使用规则

- 每一节最多一两句，不要满篇都是。
- 金句前面最好有案例，后面最好有解释。
- 金句最好能被读者单独记住、转述、引用。
- 不要硬造排比，不要为了“像金句”而空。

## 7. 结尾

### 7.1 结尾的任务

好结尾要完成三件事：

- 收束全文
- 把经验变成读者可执行的建议
- 推动读者行动或互动

### 7.2 四种结尾模板

#### 总结式

```text
回头看，这件事最重要的不是X，而是Y。
如果重来一次，我还是会先做A，再做B。
```

#### 建议式

```text
如果你也想做这件事，我建议你先从[最低成本动作]开始。
不要一开始就追求完整，先跑通第一个闭环。
```

#### 适用人群式

```text
这件事尤其适合[人群]。
如果你现在正处在[状态]，这可能是一个很好的切入口。
```

#### 互动式

```text
如果你也在做X，欢迎交流。
如果这篇文章帮你理清了一点思路，也欢迎你直接开始试。
```

### 7.3 结尾不要这样写

- 单纯喊口号。
- 空泛感谢，没有回到主题。
- 最后突然加新信息，破坏收束感。
- 只说“希望有帮助”，不告诉读者下一步该做什么。

## 最终成稿检查表

- 标题里有没有结果、对象、方法、周期中的至少两个？
- 前两段能不能让陌生读者继续读下去？
- 中段是否按问题、动作、证据、结果推进？
- 是否至少出现了几处细节、数据、案例？
- 是否有“道”和“术”，而不是只有一个？
- 是否把失败、犹豫、修正写出来了？
- 是否有一两句能被记住的判断句？
- 结尾有没有把文章收束成读者可执行的建议？

## 最短填空版

如果时间很紧，直接按这个写：

```text
标题：

一、我做了什么，结果是什么
二、我为什么开始做这件事
三、我一开始卡在哪里
四、我做对了哪几件事
五、我踩过什么坑，怎么修正
六、最后结果如何
七、这件事背后的核心经验是什么
八、如果你也想做，建议先从哪一步开始
```
