# shengcai-skill

把现有 Markdown 草稿重写成更像生财有术精华帖的 Codex Skill。

## 功能

- 输入一个已经写好的 `.md` 草稿，按生财风格重构标题、开头、正文结构和结尾。
- 适用于结果复盘型、方法教程型、趋势判断型、认知升级型、记录成长型文章。
- 保留原文事实、时间、数据、链接，不乱编结果或案例。
- 优先做结构重排、表达提纯和价值强化，而不是无依据扩写。

## 适用场景

- 你已经写完一篇草稿，但不够像“生财精华帖”。
- 你有零散的 Markdown 笔记，希望整理成可发布长文。
- 你想把经历、方法、项目复盘写得更有标题感、结构感和可读性。
- 你想让文章更贴近“龙珠文 / 生财精华文”的表达方式。

## 核心思路

这个 skill 合并了三层东西：

- `龙珠修行营`：帮你判断什么值得写
- `龙珠精华文章写作框架List`：提供复盘型文章骨架
- 大样本生财精华分析：补足标题、开头、段落推进、金句和结尾模式

最终形成一套统一流程：

`选题 -> 结构 -> 标题 -> 开头 -> 中段 -> 金句 -> 结尾`

## 快速使用

在 Codex 里直接这样用：

```text
用 $shengcai 重写这个 md 文件，让它变成更像生财精华帖的版本
```

或者：

```text
用 $shengcai 处理 /abs/path/draft.md
```

如果你给的是现成 Markdown 草稿，skill 默认会：

1. 先识别不能改的事实
2. 判断文章类型
3. 按生财模板重组结构
4. 强化标题和前两段
5. 把正文改成更清晰的编号式推进
6. 用更强的结尾收束全文

## 重写约束

- 不虚构数据、截图、客户反馈、结果
- 不擅自改日期、价格、人数、工具名、链接
- 没证据的强结论要弱化或删除
- 优先保留原稿里最有力量的句子和证据
- 输出仍然保持为 Markdown

## 参考模板

详细模板在：

- `references/ultimate-template.md`

里面包含：

- 文章类型选择
- 终极骨架
- 标题公式
- 开头模板
- 中段推进规则
- 金句句型
- 结尾模板
- 重写现有 Markdown 的检查清单

## 说明

这个仓库主要用于托管和分发 skill。

如果你想直接在本机使用，一般会把 skill 放到：

```text
~/.codex/skills/shengcai
```

然后在 Codex 中通过 `$shengcai` 调用。
